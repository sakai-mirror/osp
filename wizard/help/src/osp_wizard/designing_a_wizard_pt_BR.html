Criando um assistente

Evite complexidade desnecessária: Você pode adicionar um número ilimitado instruções, formulários, feedback, avaliação e outros recursos para cada página de um assistente. Você também pode criar um número ilimitado de páginas, categorias e subcategorias. Como resultado, é possível fazer um assistente excessivamente complexo e difícil ou pesado de usar. Lembre-se: Você não precisa usar cada elemento e característica em cada oportunidade. Se algo parece ter pouco ou nenhum valor em qualquer ponto, deixe de fora.

Assistentes podem ser muito simples ou muito complexo. Para tudo com exceção as situações mais simples, recomendamos que você projetar seu assistente antes de criá-lo em Sakai. Para projetar um assistente, siga estes passos:

  1. Analise a atividade ou processo que o assistente irá suportar. Identifique o tipo de assistente que irá suportar melhor.
  2. Se você pretende criar um assistente hierárquico, identifique as categorias e subcategorias que você vai precisar. Talvez você queira considerar a criação de uma categoria para cada caminho distinto ou agrupamento de atividades. Você pode achar útil fazer um diagrama de alto nível da organização do assistente.
  3. Identifique as páginas que você irá precisar. Talvez você queira considerar a criação de uma página para cada etapa que requer orientação diferente, produz um produto de trabalho diferente, ou ocorre em um momento diferente.
  4. Identifique o conteúdo necessário (instruções, arquivos, formulários, etc) para cada página e - se o assistente contém mais de uma página - para o assistente como um todo. Você pode achar útil para desenhar um diagrama mostrando a organização dos componentes do assistente.
  5. Se você pretende anexar arquivos para fornecer informações aos usuários do assistente, crie esses arquivos e armazená-os em Repositório.
  6. Identifique os pontos em que o feedback formativo e sumativo são necessários. Você pode permitir o feedback sobre o assistente como um todo ou em uma base página por página. Antes de tomar decisões a este respeito, pense com cuidado sobre a carga colocada nos alunos (aqueles que fornecem feedback formativo) e os avaliadores (aqueles que fornecer feedback somativo). Além disso, mantenha em mente que, depois de uma página ou um assistente foi submetido à avaliação, os participantes não podem alterá-lo.
  7. Identifique pessoas para fornecerem feedback formativo e sumativo. Certifique-se que eles têm as permissões adequadas. Para mais informações sobre como definir permissões para os revisores, consulte "Alterando permissões para a Ferramenta Assistentes".
  8. Se você planeja usar formulários para que os usuários usem em resposta a instruções em uma página do assistente e/ou para coletar comentários de revisores e avaliadores, crie esses formulários e armazene os arquivos .xsd para eles em Repositório. Formulários para coletar reflexões e os comentários de avaliação podem ser adicionados ao assistente como um todo ou para páginas individuais. Os formulários para os participantes utilizarem no acompanhamento e as instruções de preenchimento de um assistente só podem ser adicionados às páginas individuais, e não para o assistente como um todo.
  9. Escolha o estilo que pretende utilizar no assistente. Na maioria dos casos, é recomendável usar um estilo único ao longo de um assistente. Se você planeja usar um estilo diferente do padrão, armazene o arquivo .css em Repositório e use a ferramenta Estilos para criar o estilo.
