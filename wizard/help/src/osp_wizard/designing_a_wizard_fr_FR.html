Elaboration d'un assistant de saisie

Evitez toute complexité non nécessaire : Vous pouvez ajouter sans limitation des instructions, formulaires, retour d'information, évaluation, et d'autres choses encore à chacune des pages d'un assistant. Vous pouvez aussi créer un nombre infini de pages, catégories, et sous-catégories. En conséquence de quoi il est possible de faire un assistant très complexe et difficile d'abord pour le quidam. Souvenez-vous de ceci : vous n'êtes pas obligé d'utiliser chaque possibilité et élément à chaque opportunité. Si quelque chose n'apporte rien ou presque, ne l'incluez pas.

Les assistants peuvent être très simples ou assez compliqués. Dans l'immense majorité des situations, nous recommandons que vous conceviez votre assistant avant de le créer dans Sakai. Pour concevoir un assistant, suivez ces étapes :

  1. Analysez l'activité ou le processus pour lequel l'assistant apporte un soutien. Identifier le type d'assistant le plus adapté.
  2. Si vous pensez créer un assistant hiérarchique, identifier les catégories et sous-catégories dont vous aurez besoin. Vous pouvez considérer la création d'une catégorie pour chaque groupe d'activités ou cheminement distinct. Dessiner un diagramme de l'organisation de l'assistant peut vous être utile.
  3. Identifiez les pages dont vous aurez besoin. Vous pouvez considérer la création d'une page pour chaque étape nécessitant une orientation différente, qui produit un travail différent, ou qui a lieu à un moment différent.
  4. Identifiez le contenu nécessaire (instructions, fichiers, formulaires, etc...) pour chaque page et si l'assistant contient plus d'une page pour l'assistant comme un tout. Dessiner un diagramme montrant l'organisation des éléments de l'assistant peut vous être utile.
  5. Si vous prévoyez de joindre des fichiers pour apporter des informations aux utilisateurs de l'assistant, créez ces fichiers et placez-les dans les Ressources.
  6. Identifiez les moments auxquels un retour d'information est nécessaire. Vous pouvez permettre le retour d'information sur l'assistant comme un tout ou page par page. Avant de prendre une décision dans ce sens, prenez en considération la charge sur les épaules des critiques (ceux qui apportent un retour d'information constructif) et évaluateurs (ceux qui apportent un retour d'information résumé). Aussi, gardez à l'esprit que dès lors qu'une page ou un assistant a été soumis à l'évaluation, les participants ne peuvent plus le modifier.
  7. Identifiez les personnes qui apporteront un retour d'information résumé ou constructif. Soyez certains qu'ils ont les permissions appropriées. Pour plus d'informations sur les réglages de permissions pour les évaluateurs, référez-vous à "Modifier les permissions pour l'outil d'assistant".
  8. Si vous prévoyez d'utiliser des formulaires pour que les participants répondent aux instructions d'une page de l'assistant et/ou pour collecter les commentaires des évaluateurs et critiques, créez ces formulaires et placez leurs fichiers .xsd dans les Ressources. Les formulaires pour le recueil des retours d'informations et autres réflexions peuvent être ajoutés à l'assistant comme un tout ou bien page par page. Les formulaires pour l'usage des participants suivant les instructions de l'assistant peuvent seulement être ajoutés aux pages individuelles, non pas à l'assistant dans son ensemble.
  9. Choisissez le(s) style(s) que vous souhaitez utiliser dans l'assistant. Dans la majorité des cas, il vaut mieux utiliser le même style unique tout le long de l'assistant. Si vous prévoyez d'utiliser un autre style que celui par défaut, placez le fichier .css de ce style dans les Ressources et utilisez l'outil de style pour le créer.
